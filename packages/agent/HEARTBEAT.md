# GreenClaw — HEARTBEAT（心跳调度任务）

> 目标：让 GreenClaw 在无人交互时也能执行「有价值、低风险、可追踪」的自主任务。  
> 实际调度器由 `apps/backend` 实现（cron / queue / worker 均可）。

## 1. 调度原则

- 所有任务必须绑定 `project_id` 或 `tenant_id`。
- 任务结果必须可追踪：包含 `task_id`、`start_at`、`end_at`、`status`、`summary`。
- 对高成本任务设置冷却时间和幂等键，避免重复执行。

## 2. 周期任务清单

### 2.1 每日巡检（Daily Health Check）

- 频率：每天 08:30 本地时间
- 目标：
  - 检查过去 24h 失败的计算任务
  - 检查待补参数项目数量
  - 检查长时间未更新项目（>14 天）
- 产出：
  - `daily_health_report`（JSON + 人类可读摘要）
  - 可选消息推送（站内通知/邮件/企业微信）

### 2.2 每周项目周报（Weekly Project Report）

- 频率：每周一 09:00
- 目标：
  - 汇总上周项目计算次数、节能率变化、碳排趋势
  - 输出风险项（参数缺失、规范不满足、性能退化）
- 产出：
  - 周报草稿（用于前端预览）
  - PDF/Word 生成任务（可异步）

### 2.3 规则版本提醒（Ruleset Watch）

- 频率：每周三 10:00
- 目标：
  - 检查当前项目是否仍在旧规则版本下计算
  - 给出重算建议（仅建议，不自动覆盖）

## 3. 事件驱动任务

### 3.1 参数变化重算建议

- 触发条件：
  - 围护结构关键参数变化超过阈值（如外墙 U 值变化 >10%）
  - 设备系统效率参数变化（COP、热回收效率等）
- 任务动作：
  - 生成「建议重算」消息
  - 可选自动创建 `compute_job`（需用户授权）

### 3.2 报告过期提醒

- 触发条件：
  - 报告生成后关键参数再次变化
- 任务动作：
  - 标记 `report_stale=true`
  - 提示用户重新生成报告

## 4. 安全控制

- 自动重算默认关闭（`ENABLE_AUTO_RECALC=false`）。
- 心跳任务不得修改用户核心参数，仅可创建建议或草稿任务。
- 涉及外部发送（邮件、IM）必须通过租户级开关与白名单。

## 5. 推荐实现接口（后端）

- `POST /api/heartbeat/run`：手动触发某类任务
- `GET /api/heartbeat/tasks`：查询任务记录
- `POST /api/heartbeat/ack`：确认告警处理
