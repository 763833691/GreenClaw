# GreenClaw — AGENTS（OpenClaw 风格编排）

> 本文件定义 GreenClaw 的多 Agent 协作分工。  
> 实际运行时由 `apps/backend` 中的 Gateway + Agent Loop + Scheduler 执行。

## 1. Agent 角色

### 1.1 Coordinator（协调代理）

- 职责：
  - 解析用户意图（聊天或 Web UI 结构化请求）
  - 判断任务类型：参数编辑 / 快速评估 / 完整计算 / 报告生成 / 历史回溯
  - 决定调用顺序与失败回退策略
- 输入：
  - `user_message` 或 `ui_action`
  - `project_id`
  - `context`（当前参数快照、最近一次计算结果）
- 输出：
  - 标准任务计划（task plan）
  - 调用链（tools pipeline）
  - 给前端的状态说明（简短可读）

### 1.2 Calculation（计算代理）

- 职责：
  - 统一调用 `calc-engine` 的计算 API（能耗、碳排、绿建评分）
  - 处理输入参数合法性检查与单位规范化
  - 聚合多接口返回并给出统一结果对象
- 关键原则：
  - 不自行发明结果
  - 对缺失参数给出可执行补充清单

### 1.3 Compliance（规范代理）

- 职责：
  - 解释 GB/T 50378 等标准维度下的得分逻辑
  - 标注结果中「满足项/风险项/建议项」
- 输出：
  - 规则匹配说明（可用于报告正文）
  - 条文级解释摘要（仅摘要，不伪造全文原文）

### 1.4 Optimizer（优化代理）

- 职责：
  - 基于当前结果提出参数优化策略（围护、系统、运行策略）
  - 支持 “what-if” 场景，如「外墙保温+5cm」后重算
- 输出：
  - 候选优化动作列表（action list）
  - 每个动作的预期影响（能耗/碳排/绿建得分方向）

### 1.5 Reporter（报告代理）

- 职责：
  - 生成报告结构（摘要、指标表、图表说明、建议清单）
  - 调用报告生成工具输出 PDF/Word 下载地址

### 1.6 Memory（记忆代理）

- 职责：
  - 维护项目历史参数版本、历史计算任务、历史报告
  - 提供项目上下文检索（最近一次、某版本对比、关键变更点）

## 2. 工作流（聊天与 Web UI 双模式）

### 2.1 聊天模式

1. Coordinator 识别意图（问答 / 调参 / 计算 / 报告）
2. Memory 加载项目上下文（若提供 `project_id`）
3. Calculation / Compliance / Optimizer 按需执行
4. Reporter 生成可读输出（如需）
5. 返回自然语言结论 + 结构化结果摘要

### 2.2 Web UI 模式

1. 前端提交结构化请求（按钮、表单、上传）
2. Coordinator 生成任务计划并回写任务状态
3. Calculation 执行计算并产出结构化 JSON
4. Reporter 生成预览数据与文件链接
5. 前端按字段可视化（图表、表格、下载按钮）

## 3. 失败与回退策略

- 若参数不足：
  - 返回 `missing_fields`，不执行正式计算
- 若计算服务异常：
  - 记录失败任务，返回可重试状态
- 若规范解释不确定：
  - 返回「需人工复核」标志，不输出确定性断言

## 4. 心跳任务（摘要）

详细规则见 `HEARTBEAT.md`。

- 每日：检查长时间未完成任务、生成异常提醒
- 每周：生成项目周报（进展、节能趋势、风险项）
- 事件驱动：当关键参数变化超过阈值时触发重算建议
