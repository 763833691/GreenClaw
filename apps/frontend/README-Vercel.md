# GreenClaw 前端 · Vercel 部署说明（发给客户）

## 预览链接怎么用

部署成功后，Vercel 会提供一个 **Production URL**（如 `https://xxx.vercel.app`）以及每次 Push 的 **Preview URL**。用 **Chrome / Edge** 打开链接即可体验：

1. **登录**（需后端已配置演示账号或自行注册）  
2. **首页** → 新建项目  
3. **我的项目** → 进入详情、保存参数  
4. **计算中心** → 触发计算（需 Python 计算服务可达时结果更完整）  
5. **优化助手**、**报告中心** → 完成演示闭环  

若页面提示无法连接后端，请确认下方「后端地址」已在运维侧配置完毕。

---

## 关于后端地址（重要）

当前演示阶段，**API 与计算服务往往部署在临时公网地址**上，例如：

- **ngrok / cloudflared** 暴露的本机后端  
- **阿里云 / 其他云** 上的临时域名或 IP  

这些地址**后期会替换为正式域名**；替换时只需在 **Vercel 环境变量**与**后端 CORS**中同步更新，**无需改前端业务代码**（同源转发模式下）。

---

## 客户无需操作的部分

- 仓库 **Root Directory**、**环境变量** 由项目方在 Vercel 控制台配置。  
- **JWT、Cookie、HTTPS** 由后端与 Vercel 自动处理（生产环境请使用 HTTPS 与强随机 `JWT_SECRET`）。

---

## 常见问题

| 现象 | 可能原因 |
|------|----------|
| 能打开页面但接口失败 | 未配置 `VERCEL_BACKEND_ORIGIN` 或 `NEXT_PUBLIC_*` 跨域地址；后端未允许 `https://你的项目.vercel.app` |
| 登录后立刻失效 | `COOKIE_SECURE` 与域名不一致；请使用 HTTPS 预览链接 |
| 计算结果为空 | Python 计算服务未对公网暴露或未在网关配置 `CALC_ENGINE_URL` |

更完整步骤见同目录下 **环境变量清单**（由开发者在 Vercel 控制台填写）。
