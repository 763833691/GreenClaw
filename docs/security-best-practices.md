# GreenClaw 安全与权限最佳实践

## 1. 用户认证与项目隔离

- 所有业务 API 必须要求认证（JWT 或会话）。
- 项目数据必须绑定 `owner_id`（或租户 `tenant_id`），查询和更新时强制过滤。
- 禁止通过前端传入的 `owner_id` 决定归属，必须以服务端认证身份为准。

## 2. 计算沙箱建议

- 计算任务使用独立进程或容器运行，避免与网关进程共享高权限。
- 限制 CPU/内存，防止单次模拟任务拖垮系统。
- 对上传文件先扫描后解析，解析流程放在受限目录与受限权限下执行。
- 如接入 EnergyPlus，建议使用队列 + Worker，并设置任务超时和强制终止。

## 3. 文件与目录权限

- `data/` 目录仅运行用户可读写（Linux 建议 `chmod 700`）。
- 上传文件目录与导出目录分离，避免路径穿越风险。
- 禁止执行来自上传文件内容的任何脚本或宏。

## 4. API 安全

- 对登录、注册、报告生成等接口增加速率限制。
- 对文件上传接口限制：
  - 文件大小
  - 文件类型白名单（`.xlsx/.xls/.xml/.gbxml`）
  - 单用户并发数量
- 所有外部输入必须进行 schema 校验。

## 5. 密钥管理

- 生产环境不使用默认密钥（特别是 `JWT_SECRET`）。
- 模型 API Key 仅保存在服务端环境变量，不回传前端。
- 建议使用 `.env` + 服务器密钥管理（如 Vault/云平台 Secret Manager）。

## 6. 审计与可追溯

- 记录关键行为：登录、项目更新、计算触发、报告下载。
- 为每次计算生成 `job_id`，结果关联 `project_id` 与 `user_id`。
- 对 Agent 的关键结论保留引用依据（工具调用结果 ID）。

## 7. 网络与部署

- 生产环境前置反向代理并启用 HTTPS。
- 后端和计算引擎尽量置于私网，仅暴露前端入口。
- 使用防火墙限制管理端口来源 IP（SSH/监控等）。
